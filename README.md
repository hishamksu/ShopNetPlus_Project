# نظام اشتراكات العضوية المتكامل (ShopNest Plus)

مشروع متكامل لتحليل وتصميم نظام اشتراكات مدفوع لمنصة تجارة إلكترونية، تم تطويره كجزء من مقرر مادة تحليل وتصميم الأنظمة (System Analysis and Design). يهدف المشروع إلى حل مشكلة تراجع تكرار الشراء وزيادة ولاء العملاء من خلال تقديم مستويات اشتراك مخصصة توفر مزايا حصرية مثل التوصيل المجاني، الكاش باك، والعروض المبكرة.

## أهداف المشروع (Goals and Objectives)
- تعزيز ولاء العملاء: رفع معدل تكرار الشراء بنسبة تصل إلى 25% خلال سنة.
- زيادة قاعدة العملاء: استهداف جذب 50 ألف مشترك جديد في أول 6 أشهر.
- زيادة معدل الطلبات: رفع معدل تكرار الطلبات بنسبة 30% للمشتركين النشطين.

## مستويات العضوية المتوفرة (Membership Tiers)
تم تصميم النظام ليدعم ثلاثة مستويات مختلفة من الاشتراكات:
- الفئة الفضية (Silver): بسعر 29 ريال شهرياً، وتستهدف المتسوقين المتقطعين والطلاب، وتوفر توصيل عادي مجاني وخصم 5%.
- الفئة الذهبية (Gold): بسعر 59 ريال شهرياً، وتستهدف المشترين المنتظمين، وتوفر توصيل سريع خلال ساعتين وكاش باك بنسبة 10%.
- الفئة البلاتينية (Platinum): بسعر 99 ريال شهرياً، وتستهدف المحترفين وأصحاب الإنفاق العالي، وتوفر توصيل في نفس اليوم ودعم مخصص.

## نطاق النظام (System Scope)
- الخصائص المشمولة (In Scope): واجهات التسجيل والتجديد والإلغاء، لوحة تحكم المشتركين، لوحة تحكم الإدارة لتهيئة المستويات، والربط مع بوابات الدفع الإلكتروني للاشتراكات التلقائية.
- الخصائص الخارجة عن النطاق (Out of Scope): إعادة تصميم عمليات التسوق الأساسية، أو بطاقات العضوية الفيزيائية.

## النماذج والمخططات الهندسية المرفقة (UML Artifacts)
يحتوي المشروع على توثيق هندسي متكامل يغطي سلوك وبنية النظام من خلال 6 مخططات أساسية:
- مخطط حالات الاستخدام (Use Case Diagram): يوضح تفاعل العميل والإدارة مع وظائف النظام الـ 11 الأساسية.
- مخطط النشاط (Activity Diagram): يمثل تدفق خطوات عملية الاشتراك بالتفصيل.
- مخطط الكلاسات (Domain Class Diagram): يوضح البنية الهيكلية للنظام وتوزيع البيانات بين 8 كلاسات أساسية مثل User و Customer و Subscription و SubscriptionTier و Payment.
- مخطط التتابع (System Sequence Diagram): يوضح تبادل الرسائل والبيانات بين العميل والنظام أثناء عملية الدفع والاشتراك.
- مخطط حالة الكائن (State Machine Diagram): يتتبع دورة حياة الاشتراك عبر حالاته المختلفة (Pending Payment, Active, Renewal Due, Suspended, Cancelled, Expired).

## التقنيات المستخدمة (Tech Stack)
- لغة البرمجة الأساسية: Java
- إدارة قواعد البيانات: MySQL / JDBC
- أدوات نمذجة وتصميم النظام: MySQL Workbench و UML Modelling Tools

## كيف تشغل المشروع؟ (How to Run)
يتطلب تشغيل هذا المشروع إعداد قاعدة البيانات وربطها بكود الجافا عبر الخطوات التالية:

1. إعداد قاعدة البيانات:
- افتح برنامج MySQL Workbench.
- أنشئ قاعدة بيانات جديدة بنفس الاسم المستخدم في كود المشروع.
- قم باستيراد ملف الـ SQL المرفق مع المشروع لبناء الجداول والبيانات.

2. إعداد مكتبة الاتصال:
- تأكد من إضافة ملف الـ JAR الخاص بمكتبة MySQL Connector/J إلى مجلد المكتبات Libraries داخل بيئة التطوير لديك (مثل NetBeans).

3. ضبط الاتصال في الكود:
- افتح كلاس الاتصال بقاعدة البيانات (مثل DBConnection).
- تأكد من مطابقة اسم المستخدم Username وكلمة المرور Password لبيانات السيرفر المحلي بجهازك.

4. التشغيل:
- ابحث عن الكلاس الرئيسي الذي يحتوي على دالة الـ main (غالباً كلاس الواجهة الرئيسية أو Main).
- اضغط بزر الفأرة الأيمن واختر Run File لتشغيل النظام بالكامل.
